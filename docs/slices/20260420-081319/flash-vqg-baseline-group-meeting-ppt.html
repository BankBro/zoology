最朴素的理解: 模型需要记住前面写入的键值配对, 并在后面的多个查询位置正确取回对应的值。

任务本质
  • 一次前向里要做多次召回, 不是单一 query。
  • 查询位置可变, 距离可变, 需要内容相关的取回。
  • 它测的不是普通短程建模, 而是“会不会记住并正确取回”。
为什么适合 Flash-VQG
  • 既能看 local path 的近距离精确交互。
  • 也能看 remote path 的压缩存储与检索能力。
  • 非常适合判断长程瓶颈是在 reader, 还是在 memory mechanics。
第 2 章 / baseline 的结构
03 / 07

Flash-VQG 当前 baseline: 局部精确注意力 + 远程压缩记忆

Local Path

局部窗口精确注意力

  • 直接对最近窗口做精确注意力。
  • 负责短程, 细粒度, token-level 交互。
  • 当前窗口大小约为 \( \texttt{local\_num\_blocks} \times \texttt{block\_len} = 64 \) tokens。
Remote Path

码本槽级压缩记忆

  • 历史内容不会逐 token 全保留, 而是被压到若干记忆槽。
  • 每个槽同时保存 prototype 和 residual correction。
  • 读出时只保留最相关的 2 个槽。

一句话: 近处的信息精确看, 远处的信息压缩存。

第 2 章 / 数学原理
04 / 07

Flash-VQG 当前 baseline: 远程记忆的写入, 读出与融合

1. 残差表征
k̂t = VQ(kt) rt = kt - k̂t ht,s = Bs⊤ rt

先把 key 量化到码本中心, 再把残差投影到每个槽自己的局部坐标系中。

2. 写入远程 memory
gs ← γgs + Σtwt,svt Rs ← γRs + Σtwt,svtht,s⊤

当前 baseline 的写法是 weighted additive accumulation: 同时累积平均内容和细节修正。

3. 读出远程信息
Nums(qt) = gs + Rsαs(qt)

给定当前 query, 槽内 residual correction 会提供 query-conditioned 的细粒度修正。

4. 槽选择与融合
selectors(qt) = scores(qt) + log Dens(qt) ot = Fuse(otlocal, otremote)

选槽时既看匹配分, 也看槽内是否真的有足够内容; 最后 local 与 remote 在同一归一化体系里融合。

高层理解: 先把长程历史压到若干记忆槽, 每个槽同时存“平均原型”和“残差修正”, 再按当前 query 从 top-2 槽中读出, 最后与局部精确注意力统一融合。

Flash-VQG 当前 baseline = 局部精确注意力 + 远程压缩记忆

第 4 章 / 下一步实验
07 / 07

Flash-VQG: Gated-Delta Remote Memory

Before

现有 residual memory 更像“加法累计器”

  • residual 来源合理: \( r_t = k_t - \hat{k}_t \)
  • 但后续更新主要是 additive accumulation。
  • 旧内容主要只能靠统一衰减变弱, 很难定向清除。
  • 同一 code 被反复写入时, 容易出现旧信息残留和内容冲突。
After

让 memory 从“累加器”变成“可编辑状态”

  • 只更新真正命中的 code。
  • 先减去 coarse baseline, 只学习修正量。
  • Gate 决定忘多少, Delta 决定怎么改。
  • 目标是把已经写旧, 写脏, 写错的 memory 改干净。
首轮实验矩阵
additive
  • update = additive
  • forget = global
  • 作为当前 writer 对照组
gated-only
  • update = additive
  • forget = code-aware
  • 单独看 forget 是否有益
delta-only
  • update = delta
  • forget = global
  • 单独看 delta update 是否有益
gated-delta
  • update = delta
  • forget = code-aware
  • 看两者是否可叠加
主看指标
valid/accuracy acc@1024 valid/loss valid/attn/den_min valid/attn/nan_inf_count valid/vq/relative_err_mean

这一步不是推翻现有 reader, 而是先验证 remote memory 的写法, 存法和遗忘法是不是更值得优先改。
