# E5A Top2 替换审计方案

## 1. 实验定位

E5A v1 不是新的训练实验, 也不是只读已有结果表的复盘.

它的定位是:

- 第一阶段不训练新模型.
- 基于已有 checkpoint 新跑一遍带额外记录的 audit pass.
- 目标是判断: 当前 `top2` code 选择是否已经是主要瓶颈.

当前主线固定为:

- `dense-t025`
- `top2 read`
- `den_aware selector`
- `shared_den`

E5A v1 只审这条主线, 不扩展到其他 remote formula / merge mode / selector mode.

## 2. 当前主线与层范围

E5A v1 必须把 layer scope 写死.

在当前 Zoology 主线配置里:

- 模型由 `n_layers=2` 个 `TransformerBlock` 组成.
- `sequence_mixer` 使用 `Hybrid([conv, flash_vqg])`.
- `Hybrid` 按 `layer_idx % len(configs)` 选择 mixer.

因此当前主线里实际只有一个 `FlashVQGMixer`, 且它位于最后一个 `TransformerBlock` 的 `sequence_mixer` 位置.

E5A v1 默认采用单层审计:

- `audit_layer_idx = 1`, 即当前主线中唯一的 `FlashVQGMixer` 所在层.
- layer-local 指标只针对该层计算.
- `pred_mode` 表示仅在该层替换 Top2 set 后, 继续完成真实 forward 得到的最终预测.

这里的 `audit_layer_idx = 1` 是当前主线定义的一部分, 不是 E5A v1 需要 sweep 的自由参数.

若后续要研究全层同步替换或逐层 sweep, 应单独立项, 不混入 E5A v1 定义.

## 3. E5A 要回答什么

E5A 要回答的问题不是“某个新 heuristic 看起来好不好”, 而是:

> 如果只替换当前 remote read 的 Top2 code set, 而不改变当前 remote 内部近似与 local 路径, 最终输出和 MQAR 结果会不会明显更好?

如果答案是“会”, 则当前瓶颈更像 Top2 code selection.  
如果答案是“不会”, 则当前瓶颈更像 code 内近似或 remote 内部估计.

## 4. 明确不做什么

E5A v1 不做以下事情:

- 不训练新的 experiment 5 模型.
- 不修改当前训练主路径语义.
- 不引入新的 kernel 或 Triton 审计实现.
- 不把 `query` / `ref` 接进训练图.
- 不只靠已有 `history.csv` / `summary.json` / `metrics_index.json` 反推结论.
- 不把所有 token 都混进评估.
- 不把“单层审计”和“全层同步替换”混成一个定义.

E5A v1 只做:

- 读取已有 checkpoint.
- 新跑一次带额外中间量导出的 eval / audit pass.
- 构造五种 Top2 集合.
- 替换 remote 侧 Top2 集合后, 重算 layer-local audit 输出.
- 在真实 forward 中接回后续计算, 得到最终 `pred_mode`.
- 输出一组足够硬的聚合指标表.

## 5. 当前机制的正确表述

不要把当前机制写成:

> “先选 top2, 再在 top2 内单独归一化, 得到 remote 输出, 然后再和 local 合并.”

更准确的表述是:

1. 先为当前 row 选出 Top2 code.
2. remote 分支只保留这两个 code 的贡献.
3. local 分支保持不变.
4. remote 与 local 一起进入最终 shared denominator 合并.

还必须注意一点:

- remote 与 local 在 phase2 中共用同一个数值稳定化最大值 `m_shared`.
- 这个 `m_shared` 不是只在 Top2 内重新计算的局部 max, 而是当前 row 上 remote 与 local 共同使用的公共 max.
- E5A v1 若做 runtime override, 只能替换 Top2 code set, 必须沿用同一个 `m_shared` 机制, 不能改成“只在被选中的 Top2 内重新做一遍 softmax”.
- `mode` 只负责决定“选哪些 code”, 不直接手工指定新的 read 权重.
- 一旦 Top2 code set 被替换, 被选中的 code 会继续按当前主线 forward 的既有规则自然得到读出权重, 未选中的 code 权重则变为 0.

在当前 `clr_v1 + shared_den` 主线里, 更贴实现的底层量是:

- `Num_far`
- `Den_far`
- `Num_loc`
- `Den_loc`

最终输出是:

$$
h = \frac{Num_{far} + Num_{loc}}{Den_{far} + Den_{loc}}
$$

其中 remote 权重的稳定化应沿用:

$$
m_{shared} = \max(m_{remote}, m_{local})
$$

也就是说, remote 路径的权重计算应保持为 `\exp(logit - m_{shared})`, 而不是在被选中的 Top2 内重新定义一个新的归一化基准.

若把 mode 对应的 Top2 集合写为 $\mathrm{Top2}_{mode}(r)$, 则更贴实现的有效读出权重可以理解为:

$$
w^{eff}_{mode}(r,s) = w_{base}(r,s) \cdot \mathbf{1}[s \in \mathrm{Top2}_{mode}(r)]
$$

这里的含义是:

- `w_base` 仍按当前主线的既有读出规则计算.
- `mode` 改变的是保留哪些 code.
- E5A 不在新 Top2 子集内额外重新定义一套权重公式.

因此 E5A 中的 `o_remote_audit` 不是模型长期保存的新状态, 而是 audit-only 派生量:

$$
o^{remote}_{audit} = \frac{Num^{far}_{audit}}{Den^{far}_{audit} + \varepsilon}
$$

## 6. 五种 Top2 集合

E5A v1 固定审以下五种 Top2 集合:

- `student`
- `write`
- `time`
- `query`
- `ref`

其中:

- `student`: 当前模型真实 forward 中实际使用的 Top2.
- `write`: 只看 write-side 质量分布.
- `time`: 在 write-side 基础上再考虑时间项.
- `query`: 看 query-key 匹配驱动下哪些 code 更该被选.
- `ref`: 在 query-key 匹配和时间项同时成立时, 哪些 code 更该被选.

`ref` 是 E5A 中最接近 oracle 的 audit-only 集合, 但不是训练目标.

## 7. row 集合与 remote-visible scope

MQAR 中只有真正 query 位置的标签不是 `-100`, 其余位置都应忽略. 定义:

$$
\text{task\_rows} = \{ r \mid y_r \neq -100 \}
$$

其中:

- 任务 accuracy
- `pred_changed_vs_student`

都只在 `task_rows` 上统计.

机制审计再进一步收紧为:

$$
\text{audit\_rows} = \text{task\_rows} \cap \{ r \mid J_r^{remote} \neq \varnothing \}
$$

其中:

- Top2 overlap
- `ref_mass_capture`
- `top1_top2_margin`
- `remote_dist_to_ref`
- `final_dist_to_ref`

都只在 `audit_rows` 上统计.

设输入按 block 组织为 `[B, H, N, L, ...]`, 其中 $T = N \cdot L$.  
当前主线 remote path 的因果边界定义为:

$$
idx_{\text{remote}}(n) = \max(0, n - (w - 1))
$$

其中 $w = local\_num\_blocks$.

因此 block $n$ 内任一 row 的 remote-visible history 终点为:

$$
hist\_end(n) = idx_{\text{remote}}(n) \cdot L
$$

对应的 remote-visible causal history 是:

$$
J_r^{remote} = \{ j \mid j < hist\_end(n) \}
$$

`write / time / query / ref` 四种 score 都必须在同一个 $J_r^{remote}$ 上定义.

## 8. 统一的 code score 定义

后文统一使用 $z^{mode}_{r,s}$ 表示 row $r$ 对 code $s$ 的审计分数, 其中 $mode \in \{write, time, query, ref\}$.

### 8.1 Write

$$
z^{write}_{r,s}
=
\log \sum_{j \in J_r^{remote}} \pi_{j,s}
$$

### 8.2 Time

$$
z^{time}_{r,s}
=
\log \sum_{j \in J_r^{remote}} \pi_{j,s} \exp(c_r - c_j)
$$

### 8.3 Query

$$
z^{query}_{r,s}
=
\log \sum_{j \in J_r^{remote}} \pi_{j,s} \exp(q_r^\top k_j)
$$

### 8.4 Reference

$$
z^{ref}_{r,s}
=
\log \sum_{j \in J_r^{remote}} \pi_{j,s} \exp(q_r^\top k_j + c_r - c_j)
$$

其中:

- $\pi_{j,s}$ 对应 weighted routing write 的 `weights[..., s]`.
- $q_r$ 是当前 row 的 query 向量.
- $k_j$ 是历史 token $j$ 的 key 向量.
- $c_r, c_j$ 是 CLR path 的时间项.

因此要按上式精确计算 `z_write / z_time / z_query / z_ref`, 除了当前文档已列出的 `Q_blk`, `c_q`, `c_end`, `hist_end` 之外, 还必须拿到:

- write-side routing 权重, 即 `\pi_{j,s}` 对应的 `weights`.
- 历史 token 级 key 张量, 即 `K_blk` 或等价物.
- 历史 token 级时间张量, 即 `c_all` 或等价物.

## 9. 从层内审计到最终预测

E5A 必须区分两类输出.

### 9.1 层内审计输出

对任意候选 Top2 集合, 定义:

$$
o^{remote}_{audit} = \frac{Num^{far}_{audit}}{Den^{far}_{audit} + \varepsilon}
$$

$$
h^{layer}_{audit} = \frac{Num^{far}_{audit} + Num^{loc}}{Den^{far}_{audit} + Den^{loc} + \varepsilon}
$$

其中:

- `o_remote_audit` 只用于 remote-level 误差比较.
- `h_audit_layer` 只用于 layer-local audit.

### 9.2 模型级最终输出

`h_audit_layer` 不能直接等同于最终任务预测.  
`pred_mode` 的来源必须是:

1. 在真实 model forward 中进入 `audit_layer_idx`.
2. 用 mode 对应的 `top2_codes` 替换当前 student Top2.
3. 用替换后的 remote contribution 形成新的该层输出 `h_audit_layer`.
4. 将该层输出继续送入模型剩余未改动的计算.
5. 最终经 lm head 得到 `logits_mode`.
6. 定义:

$$
pred^{mode} = \arg\max logits^{mode}
$$

7. 再在 `task_rows` 上计算 accuracy.

必须明确:

> 替换 Top2 set 的动作发生在真实 forward 内, 然后继续完成后续计算得到最终预测; `h_audit_layer` 只用于机制指标, 不直接等同于任务预测.

## 10. 所需中间量与责任边界

E5A 需要大量逐行, 逐头, 逐 code 的中间张量.  
现有 scalar metrics 链路不足以承载这些张量:

- `FlashVQGMixer.get_scalar_metrics()` 只保留标量.
- `Trainer` 只汇总标量 metrics.

因此 E5A v1 需要额外提供 audit-only tensor capture helper.

### 10.1 `Flash-VQG` 负责导出的原始张量

必需导出:

- `Q_blk`
- `K_blk`, 或等价的 token 级 history key 张量
- `write_weights`, 即 RoutingVQ `weights` 对应的 $\pi_{j,s}$
- `c_all`, 或等价的 token 级时间张量
- `c_q`
- `c_end`
- `student_top2`
- `student_top1_top2_margin`
- `Num_loc`
- `Den_loc`
- `local_max`
- `m_shared`, 即当前 row 上 remote / local 共用的稳定化 max
- `g_remote`
- `R_remote`
- `L_remote`
- `a_remote`
- `codebook`
- `clr_basis`
- `hist_end`

可选导出:

- `student_top2_logits`

不要求导出:

- 全 code 的 `student_logits`

### 10.2 `zoology` 负责构造和计算的量

由 `zoology` 侧负责:

- `task_rows`
- `audit_rows`
- `z_write`
- `z_time`
- `z_query`
- `z_ref`
- `top2_write`
- `top2_time`
- `top2_query`
- `top2_ref`
- 聚合指标表与 `summary.json`

其中:

- `z_write / z_time / z_query / z_ref` 在 `zoology` 中基于 `write_weights`, `K_blk`, `c_all`, `Q_blk` 和 `hist_end` 计算.
- `c_q / c_end` 继续保留, 主要用于和当前 remote path 的边界与时间偏置保持一致.
- layer-local 的 `o_remote_audit / h_audit_layer` 若在 `zoology` 侧重算, 必须复用同一个 `m_shared`, 不能对 override 后的 Top2 子集单独重做局部 softmax.

### 10.3 责任边界

最合适的边界是:

- `Flash-VQG`: 导出张量与 runtime override 所需 helper.
- `zoology`: 基于这些张量完成 E5A 指标计算, 汇总和实验输出.

## 11. override 机制与实现约束

E5A v1 不要求引入训练主路径的永久配置项.

更推荐的做法是:

- 在 Flash-VQG attention 层实现 audit-only runtime helper 或 hook.
- 当 audit pass 开启时, 指定层临时接收外部 `top2_codes` override.
- 默认训练与普通 eval 路径完全不受影响.

当前 Zoology 主干 forward 不沿整条模型链显式透传 audit 参数, 因此 E5A v1 更推荐采用模块级 runtime 上下文触发, 而不是把 audit 开关做成训练配置项或新增一整条 forward 参数链.

推荐的最小接口形态是:

- 在 `FlashVQGMixer` / `FlashVQGAttention` 上提供 `set_audit_runtime(runtime)` 与 `clear_audit_runtime()`.
- `runtime` 只承载本次 audit forward 的临时控制信息, 例如:
  - `enabled`
  - `capture_tensors`
  - `mode_name`
  - `override_top2_codes`
  - `capture_sink`

触发方式应为:

1. `e5a_audit.py` 在进入目标层 forward 前临时设置 `runtime`.
2. 跑一次正常 model forward.
3. forward 结束后立即清理 `runtime`.

`FlashVQGAttention` 在 forward 内对 `runtime` 的使用约束是:

- 若 `runtime is None` 或 `runtime.enabled == False`, 必须完全走现有 baseline 路径.
- 若 `runtime.capture_tensors == True`, 只额外导出审计所需中间张量.
- 若 `runtime.override_top2_codes` 非空, 只替换 remote 侧 Top2 保留集合.
- 不得借机改动 `m_shared`, read 权重公式, local path, shared denominator merge 或其他默认计算路径.

E5A v1 的实现约束是:

- 禁止 row-wise / code-wise Python loop.
- 优先采用张量化实现.
- 允许沿 block, history, batch 或 case 维做 chunked vectorization.

具体建议:

- `write / time`: 优先 prefix snapshot.
- `query / ref`: 优先 blockwise vectorization.
- 当全量 `[B,H,N,L,T]` 张量过大时, 允许沿 `T` 或 batch 维做 chunked vectorization.

这不改变数学定义, 只改变工程实现形态.

- `512x128`
- `1024x256`

所有 `acc_*` 指标都必须与现有 Zoology `valid/accuracy` 保持同一聚合口径:

1. 对每个样本, 仅在该样本的 `task_rows` 上计算 token-level mean accuracy.
2. 再对样本做均值聚合.

也就是说, `acc_total`, `acc_512x128`, `acc_1024x256` 都采用 per-example mean 再 overall mean, 而不是把所有有效 row 直接混成一个 row-weighted mean.

## 13. 裁决规则

### 13.1 定义 student 明显偏离 ref

满足任一条即可:

- `overlap_vs_ref(student) <= 0.60`
- `ref_mass_capture(student) <= 0.75`

### 13.2 定义 ref 替换带来有意义提升

在核心 case `512x128` 和 `1024x256` 上同时满足:

- 平均 `acc_ref - acc_student >= 1.0pp`
- 任一单 case 不低于 `0.5pp`
- 不存在单 case 回退超过 `0.3pp`

### 13.3 定义 ref 替换提升有限

同时满足:

- 平均 `acc_ref - acc_student < 0.3pp`
- 两个核心 case 的增益都小于 `0.5pp`

### 13.4 定义 student 已经比较接近 ref

同时满足:

- `overlap_vs_ref(student) >= 0.80`
- `ref_mass_capture(student) >= 0.90`

### 13.5 最终裁决

判定 `selection_bottleneck`:

- 满足 13.1
- 且满足 13.2

判定 `in_code_approx_bottleneck`:

- 满足 13.4
- 且满足 13.3

其他情况统一归为:

- `mixed_or_inconclusive`

`remote_dist_to_ref` 与 `final_dist_to_ref` 作为佐证项使用, 不单独设置绝对裁决阈值.

## 16. 一句话执行摘要

E5A v1 是一个不训练的 Top2 替换审计实验.  
它固定当前主线 checkpoint, 在当前主线中唯一的 `FlashVQGMixer` 上构造 `student / write / time / query / ref` 五种 Top2 集合, 并在不改变 local 路径与 remote 近似公式的前提下, 只替换 remote 侧 Top2 code set, 重算 `o_remote_audit` 与 `h_audit_layer`, 再接回真实 forward 得到 `pred_mode`.  
通过 `overlap_vs_ref`, `ref_mass_capture`, `top1_top2_margin`, `remote_dist_to_ref`, `final_dist_to_ref` 与 MQAR accuracy 的联合变化, 判断当前瓶颈更像 Top2 code selection, 还是更像 code 内近似问题.

## 18. 本轮实验结论

结合主线实验规划文档, E5A v1 正式结果, 以及本轮 dense teacher 扩展实验, 当前结论可以收口为三层.

第一层是主线层面的结论, 这一点没有变化:

- 实验 1 到实验 3 仍然支持继续把 `dense-t025 + top2 read + den_aware selector + shared_den` 作为当前默认工作点.
- 其中:
  - `top2 read` 相对 `dense read` 的收益已经在主线实验中成立.
  - `den_aware selector + shared_den` 没有被 denominator-free 方案稳定打败.
  - `dense-t025` 相对 `dense-t050` 的 matched validation 已完成收口, 且长序列收益最稳定.
  - `cb128` 仍是当前甜点, 单纯扩大 codebook 容量不是优先方向.

第二层是 E5A v1 的否定性结论, 这一点也仍然成立:

- 基于 `dense-t025` 主线的两个 seed(`s123` 与 `s124`)的正式 E5A run, 当前结论不支持“只替换 remote read 的 Top2 code set 就能明显提升最终输出和 MQAR accuracy”.
- 两个 run 的 `final_ruling` 都是 `mixed_or_inconclusive`, 没有任何一个 run 满足 `selection_bottleneck`.
- `ref` 作为当前 audit 定义下最接近 oracle 的 audit-only 选集, 在两个 run 上都显著劣于 `student`.
  - `e5a-run-001`: `acc_ref - acc_student = -14.33pp`, 其中 `512x128 = -10.70pp`, `1024x256 = -17.96pp`.
  - `e5a-run-002`: `acc_ref - acc_student = -17.85pp`, 其中 `512x128 = -11.20pp`, `1024x256 = -24.49pp`.
- `query` 与 `ref` 在两个 run 上几乎重合, 说明当前定义下 `time` 项几乎没有改变最终的 Top2 集合, 主导信号基本就是 query-key matching.
- `write` 与 `time` 两种 mode 在两个 run 上都接近完全崩溃, 说明仅靠 write-side统计量不足以恢复当前模型真正需要的 remote code set.

第三层是 dense teacher 扩展实验带来的新结论, 这是本轮新增的信息:

- `Attention-Mass Teacher`(`dense_mass`) 与 `ref` 在两个 seed 上完全重合.
  - `s123`: `acc_dense_mass = acc_ref = 0.786941`
  - `s124`: `acc_dense_mass = acc_ref = 0.755156`
  - 这说明此前失败的 `ref` 本质上已经等价于“exact dense token attention mass 聚合到 code”的 teacher. 仅靠 attention mass 本身, 依然救不回 student.
- `Value-Contribution Teacher`(`dense_value`) 在两个 seed 上都显著优于 `query/ref`, 而且提升主要体现在真正困难的 `1024x256`.
  - `s123`: `acc_dense_value = 0.895859`, 相对 `query +10.51pp`, 相对 `ref +10.89pp`, 但仍比 `student` 低 `3.44pp`.
  - `s124`: `acc_dense_value = 0.915402`, 相对 `query +15.98pp`, 相对 `ref +16.02pp`, 但仍比 `student` 低 `1.82pp`.
  - `1024x256` 上的修复尤其明显:
    - `s123`: `0.693211 -> 0.820867`, `+12.77pp`
    - `s124`: `0.631086 -> 0.845195`, `+21.41pp`
- `dense_value` 还显著降低了“把原本正确样本改坏”的比例.
  - `s123`: `pred_changed_vs_student` 从 `ref` 的 `19.54%` 降到 `6.93%`
  - `s124`: `pred_changed_vs_student` 从 `ref` 的 `24.93%` 降到 `5.73%`
  - 其中 `student correct -> mode wrong` 也显著下降:
    - `s123`: `16.62% -> 4.75%`
    - `s124`: `20.99% -> 3.28%`

因此, 需要把当前结论进一步修正为:

- 不能把当前主瓶颈直接裁成 `selection_bottleneck`, 因为即使使用更强的 `dense_value` teacher, post-hoc 单层 Top2 硬替换仍没有超过 `student`.
- 但也不能再简单地说“selection 没什么信号”. `dense_value` 相对 `query/ref` 的稳定大幅提升说明, teacher 定义确实很重要, 而且 selection 里存在真实的 value-aware signal.
- 更准确的判断是:
  - `ref` 这类 attention-mass / code-level heuristic teacher 太浅, 不能代表真实 end-to-end 有用的 selection signal.
  - `dense_value` 捕获到了更接近最终 remote value 贡献的 teacher, 因而能显著修复 `ref/query` 的失败.
  - 但 `student` 依然最好, 说明当前模型里的 selection 仍然与 codebook/read approximation/shared_den/downstream interpretation 强耦合, 不能被单层 post-hoc 硬替换直接兑现.

基于这些结果, 当前最合理的总判断是:

- 当前系统的瓶颈不是“纯 selection 错了”, 也不是“selection 完全不重要”.
- 更像是:
  - selection signal 确实存在, 而且需要 value-aware teacher 才能暴露出来;
  - 但这类 signal 还不能脱离现有表示层和 downstream 共适应而独立替换.

因此, 本轮实验的最终结论是:

- 当前 baseline 没有被推翻.
- `ref` 失败这一事实仍然有效, 但它更像是否定了“attention-mass teacher 足够好”, 而不是彻底否定 selection 方向.
- `dense_value` 的结果已经提供了新的正向证据: teacher 重定义是有价值的, 但现在还不足以直接进入实验 5 训练.
- 当前最合理的下一步不是立刻训练, 而是进入阶段 B: 以 `dense_value` 为 teacher 做 partial override 和 margin / head sensitivity 补分析, 再决定后续优先走实验 4 还是实验 5.

## 19. 下一步实验建议

### 19.1 总体原则

下一步建议采用阶段式推进:

1. 先做路线分流实验.
2. 再做补审计解释结果.
3. 最后只选择一条训练主线.

当前不建议直接推进实验 5, 也不建议同时铺开实验 4 和实验 5.

### 19.2 阶段 A: 先做路线分流

阶段 A 的唯一主分流实验应是 `token-level exact dense remote teacher audit`(下文简称 `dense teacher audit`).

它的定义是:

- 不再直接使用当前 `query/ref` 这类 code-level heuristic.
- 也不是重复导出当前 student selector 已在使用的 code-level Top2 logits.
- 改为先在 token-level 上计算 remote-visible token 的 dense 匹配证据, 再结合 `write_weights(j,s)` 聚合到 code 维, 得到新的 teacher Top2.

一个可接受的 teacher 形式是: 对 row `r` 和 code `s`, 用 remote token `j` 的 `write_weights(j,s)` 聚合 `q_r · k_j - c_j` 这类 token-level dense remote 证据, 构造 `z_teacher(r,s)`.

为避免和当前 E5A 已有的 `student / query / ref` 混淆, 可以把四者的差异整理如下:

| 对象 | score 从哪里来 | 在哪一层聚合 | 它到底在回答什么问题 |
| --- | --- | --- | --- |
| `student` | 来自模型真实 forward 中当前 selector 的打分. 在当前主线下, 本质上是 code-level selector logits, 约等于 `q · codebook + p_bias + log(den_eff)` 这一路 | 直接在 `code` 层排序并取 Top2 | 当前模型线上真实会选哪 2 个 code |
| `query` | 来自 E5A audit 侧重算的 query-driven code score. 先看 query 对 remote 信息的匹配, 再映射成 code 分数 | 已经是 `code` 层分数, 然后直接取 Top2 | 如果只按 query-key 匹配驱动, 哪些 code 更该被选 |
| `ref` | 来自 E5A audit 侧重算的 reference code score. 比 `query` 多带一点时间/边界校正项, 但仍然是 audit 定义的 code-level heuristic | 已经是 `code` 层分数, 然后直接取 Top2 | 如果按当前 audit 定义下最接近 oracle 的 code-level heuristic, 哪些 code 更该被选 |
| `new dense teacher` | 不直接从 code-level heuristic 来. 先看每个 query row 对所有 remote-visible token 的 token-level exact dense remote 证据, 再结合 `write_weights(j,s)` 聚合到 code 维 | 先在 `token` 层累计证据, 再聚合到 `code` 层后取 Top2 | 如果 teacher 不再靠手工 code-level heuristic, 而是直接从更底层的 dense remote token 证据聚合出来, 哪些 code 才真正更该被选 |

当前最值得先实现的 dense teacher 候选可收口为两个:

1. `Attention-Mass Teacher`

设 $\alpha_{dense}(r,j)$ 为 row $r$ 在 remote-visible token $j$ 上的 exact dense remote attention 权重, $\pi(j,s)$ 为 token $j$ 写到 code $s$ 的权重. 定义:

$$
z_{mass}(r,s)=\sum_{j \in J_r^{remote}} \alpha_{dense}(r,j)\,\pi(j,s)
$$

再按 $z_{mass}(r,s)$ 在 code 维取 Top2.

这个候选的优点是实现最直接, 与当前 E5A payload 也最容易对齐. 它更像当前 `ref` 的严格版对照, 主要回答“当前 `ref` 与 exact dense token attention mass 聚合到 code 之后到底差多远”.

2. `Value-Contribution Teacher`

先定义每个 code 对 exact dense remote output 的分量:

$$
u(r,s)=\sum_{j \in J_r^{remote}} \alpha_{dense}(r,j)\,\pi(j,s)\,v_j
$$

再定义 exact dense remote output:

$$
o_{dense}(r)=\sum_{j \in J_r^{remote}} \alpha_{dense}(r,j)\,v_j
$$

然后用每个 code 对最终 dense remote output 的对齐程度作为分数:

$$
z_{value}(r,s)=\langle u(r,s), \hat{o}_{dense}(r)\rangle
$$

其中 $\hat{o}_{dense}(r)$ 表示对 $o_{dense}(r)$ 做 $L2$ 归一化后的方向向量. 最后按 $z_{value}(r,s)$ 在 code 维取 Top2.

这个候选比 `ref` 更不一样, 更接近 end-to-end 语义. 它更直接回答“当前 selection 应该追的到底是 attention mass, 还是对最终 remote value 的真实贡献”.

当前推荐顺序是:

- 若只想做一个便宜的 sanity control, 可以先做 `Attention-Mass Teacher`.
- 若只保留一个正式主分流 teacher, 优先做 `Value-Contribution Teacher`, 因为它相对当前 `ref` 的新增机制分流价值更高.

它要回答的问题是: 如果把更像 token-level exact dense remote teacher 的 Top2 塞回当前 forward, 最终预测会不会变好.

阶段 A 的裁决规则是:

- 若 dense teacher override 仍明显劣于 student, 则说明问题不只是当前 `query/ref` 定义不够准, 更像“post-hoc Top2 硬替换本身不成立”.
- 若 dense teacher override 明显优于当前 `query/ref`, 甚至接近或超过 student, 则说明 selection 仍值得继续做, 只是 teacher 需要重定义.

阶段 A 可以并行一项低成本补分析:

- 基于已有 `row_audit.parquet` 做精简版 error taxonomy, 只看 `1024x256`、`student correct -> mode wrong`、以及低 `overlap_vs_student` 的 rows.

### 19.3 阶段 B: 再做耦合性验证

只有在阶段 A 完成后, 才建议进入阶段 B. 阶段 B 的目标是验证“selection 改动为什么会把已有正确样本改坏”, 即测耦合强度.

最小建议做 3 类 partial override:

- 只在 student 与 teacher disagreement 的 rows 上替换.
- 逐 head 替换, 而不是一次替换整层所有 head.
- 只在高置信 teacher rows 上替换.

若 partial override 明显好于整层硬替换, 更支持“问题在共适应被破坏”. 若 partial override 也普遍变差, 更说明 selection 不是当前最优先的瓶颈定位方向.

阶段 B 还建议追加一类辅助 telemetry:

- student 与 mode 最终输出的 top1-top2 logit margin.

它用于区分: 坏样本是本来就靠近决策边界, 还是 override 真把 representation 推离了 downstream 已学到的语义流形.

### 19.4 阶段 C: 只保留一条主线进入新训练

阶段 C 不应同时推进实验 4 和实验 5, 而应根据阶段 A/B 的结果二选一.

优先走实验 4 的条件:

- dense teacher override 也无法带来 end-to-end 改善.
- partial override 只能减少伤害, 但不能给出明确收益.
- 错误分析显示问题更像“现有 code 表示或 code 内近似不足”, 而不是“teacher-guided selection 还没学到”.

此时更合理的主判断是: 当前瓶颈更接近单级 quantization / code 内表征能力, 后续优先方向应转向实验 4 的两级 key quantization.

优先走实验 5 的条件:

- dense teacher override 相对当前 `query/ref` 有明显改善.
- partial override 至少在部分 row/head 上能稳定减少错误.
- 这表明 selection 本身仍有可学空间, 只是当前 student selector 没有学到合适的 teacher 对齐.

此时更合适的动作是: 以 token-level exact dense remote teacher 或其蒸馏版本重新定义 selector supervision, 让 selector 与 downstream 共同重新适应.

### 19.5 当前推荐执行顺序

基于当前证据, 推荐执行顺序固定为:

1. 先做 `dense teacher audit`.
2. 同步做精简版 row-level error taxonomy.
3. 再做 partial override 与 margin 补分析.
4. 最后只在实验 4 和实验 5 中选择一条主线进入训练.

其中可以并行的只有 `dense teacher audit` 与精简版 error taxonomy.

不建议并行铺开的包括:

- 实验 4 与实验 5.
- 多种 teacher 定义的大规模同时试错.
- 在没有 dense teacher 结论前, 直接进入新训练实验.

若只保留一个最高优先级动作, 则应明确写为:

- **先做 dense teacher 版 E5A.**

因为这一步将直接决定:

- 后续应继续把问题视为 selector teacher / retrieval target 定义问题.
- 还是应转向更偏表示层的瓶颈, 即实验 4 所关注的量化级数与 code 内近似能力问题.
