# gd_residual_v1 `mu_min_count` 50 batch 细化报告

日期: 2026-04-26

## 1. 本轮结论

本轮 50 batch 细化实验支持以下判断:

- `mu_min_count=1.0/0.5` 在 50 batch 后仍然 `mu_valid_ratio=0.0`, 继续扩大 short-run 没有优先级, 只应保留为对照.
- `mu_min_count=0.25` 只在后期出现极低比例非零, two-seed final valid mean 约 `0.004`, 不够稳定.
- `mu_min_count=0.2` 稳定非零, final valid mean 约 `0.109`, 但有效比例偏保守.
- `mu_min_count=0.15` 稳定非零且跨 seed 对齐, final valid mean 约 `0.497`, 是本轮最平衡候选.
- `mu_min_count=0.1` 最稳定且比例最高, final mean 约 `0.743`, 但 50 batch 下 valid loss 未显示优于 `0.15/0.2/1.0`.

因此本轮不声称低阈值带来任务质量收益. 它们明确解决了 `mu_valid_ratio=0` 问题, 但 valid loss 基本相同. 下一步建议进入 `100 batch + baseline` 对照, 候选为 `0.15` 和 `0.1`; 若只能选一个更保守阈值, 优先 `0.15`. `0.2` 可作为保守备选, 但不建议占用主实验名额.

## 10. 阈值对比

- `1.0/0.5`: 50 batch 后 final valid `mu_valid_ratio=0.0`, 且 `L_state_max` 仍远低于 0.5. 这两档确认过强, 不再作为候选扩展, 只保留做对照.
- `0.25`: final valid 有非零, 但比例过低且 seed 间差异明显, two-seed final valid mean 约 `0.004`. 它没有解决“稳定有效”的问题.
- `0.2`: 从 step20 起稳定非零, final valid mean 约 `0.109`. 这是保守可行阈值, 但有效覆盖面偏小, 不建议作为主候选优先于 `0.15`.
- `0.15`: step0 到 step50 都在约 `0.47-0.52`, 跨 seed 稳定, `m_norm_max` 未见持续放大, 无 NaN/Inf. 这是本轮最合理的保守候选.
- `0.1`: 有效比例最高且稳定在约 `0.743-0.746`, 无 NaN/Inf, `m_norm_max` 未见放大. 但 valid loss 未优于 `0.15`, 因此只能作为“高覆盖候选”, 不能直接定为默认.

## 11. 判断标准对照

- A: `0.15` 和 `0.1` 满足稳定非零, valid loss 不差于 `1.0`, `m_norm_max` 未持续放大, inject_ratio 非零, 无 NaN/Inf. 列为 100 batch 候选.
- B: `0.15` 达到非零且稳定性接近 `0.1`, 因此优先推荐更保守的 `0.15`, 不直接选择 `0.1` 作为唯一候选.
- C: 不适用, 因为 `0.15/0.2` 都不是接近 0.
- D: 不适用, 低阈值没有让 valid loss 明显变差, `m_norm_max` 也没有明显放大.
- E: 命中. `0.1/0.15/0.2/0.25` 的 valid loss 与 `1.0` 基本一样, 不能声称低阈值有质量收益, 只能说低阈值解决了 `mu_valid_ratio=0`.
- F: 命中. `1.0/0.5` 在 50 batch 后仍为 0, 停止继续扩大它们的 short-run, 除非作为最终对照.

## 12. 下一步建议

进入下一轮 `100 batch + baseline` 对照, 但不要启动最终正式实验. 建议矩阵:

| run | 目的 |
| --- | --- |
| baseline / local_only 或既有 `legacy_fox` 对照 | 判断低阈值是否带来任务收益, 不是只看 gd residual 内部指标 |
| `gd_r16_wk4, mu_min_count=0.15` | 主候选, 更保守且稳定非零 |
| `gd_r16_wk4, mu_min_count=0.1` | 高覆盖候选, 检查是否带来收益或 noisy target 代价 |
| 可选 `gd_r16_wk4, mu_min_count=0.2` | 若资源允许, 作为更保守低覆盖 guardrail |

下一轮继续保留 step0/10/20/... valid 和 diagnostics. 若 100 batch 下 `0.15/0.1` valid loss 仍与 baseline 基本一样, 不应进入正式实验, 应转向 residual target 质量或 single-batch state/mask debug. 若其中一档 valid loss 不差且开始优于 baseline, 再考虑扩大 seeds 或进入性能优化前的更长 short-run.
